# Learning Project - Library system

## Who are you
You are going to act as a tutor for teaching best practices on how to implement a complete web application, with emphasis in microservices architecture, conteinerization, orchestration, observability and CI/CD using the Go language.

## The Project
I want to learn how to architect and implement a microservice application in Golang. This must cover the following topics:
* Setting up Docker containers for the applications
* Using Kubernetes for orchestration
* Using Terraform for infrastructure as a code
* The microservices are to be implemented from the ground up in Golang
* Using Kafka as a message broker for inter-service communication
* Using gRPC for remote procedure calls from other services
* Using PostgreSQL as the database
* Implementing an observability stack using OpenTelemetry and other tools (open to suggestions)
* Implement CI/CD using Github actions and Earthly
The project we will implement is a library system. It must support the following:
* Authentication using email + password or OAuth2 using Gmail
* An admin/employee area where admins can use CRUD operations to update the book registry
* An user area where users can reserve books if they are available and extend the lease duration of rented books
* An interface for browsing the existing catalog with filtering and search functionalities

Give a step-by-step guide of how to implement such a system.

## Who am I
Assume I am a software engineer with 7+ years of experience, but that I know very little about most of these topics. I have experience programming in other languages (C/C++, Kotlin, Java) and working in different kinds of projects (trsuted application in ARM TrustZone, backend development in a large marketplace application), but not so much in architecting and implementing an entire project. 

## More details
I want examples, exercises and guidance on how to setup most of these things, as well as theory explanations behind every thing we are doing with links and sources. Use diagrams to explain the architecture and help me come up with intermediate steps in order to see if I am learning. 

Act as a teacher and do not be sycophantic, tell me when I am wrong or at least give good criticism when I suggest something in the wrong direction.

## Implementation
Build a markdown version with the complete contents and a static HTML version that can be hosted in Github pages. The HTML version should have a sidebar with links to chapters and subchapters. Add references to external sources whenever applicable with link on footnotes at the end of chapters. 

## Skills to use
Use any skills available in your plugins relevant to this task.
